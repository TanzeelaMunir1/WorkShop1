Git Commands

Version command

By git --version command , the git version can be check.

Help

By git --help command , all basic git commands can be show with description.

Init

By git init command a new Git repository is created. It can be used to convert an existing, project to a Git repository or initialize a new, empty repository

Status

By git status command the state of the working directory display and the staging area. It lets you see which changes have been staged, which haven't, and which files are tracked by Git and which are not.

Add

git add . command adds a change in the working directory to the staging area. It tells Git that you want to include updates to a all file in the next commit.

git add FileName(test.file) command adds a change in the working directory and updates to a particular file in the next commit.

rm cached

git rm --cached flag removes a file from the staging area. The files in the working directory will remain. This means that you'll still have a copy of the file locally but the file will be removed from the tracking on your Git project.

Clone Command

git clone is primarily used to point to an existing repository and make a clone or copy of that repository at in a new directory and at another location. The original repository can be located on the local file system or on remote machine and accessible supported protocols.The git clone command copies an existing Git repository.

Push Command

git push command is used to upload local repository content to a remote repository. Pushing is how you transfer commits from your local repository to a remote repository.
